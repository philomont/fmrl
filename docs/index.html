What is FMRL?

.fmrl (Fragile Manuscript Record Layer) is an image codec where visual degradation is a core design feature. Images decay over time and with repeated access — simulating the natural aging of physical media. This is intentional art/archival design, not a bug.

  • Temporal decay — images fade based on days since last view
  • Usage-based wear — edge erosion accumulates with each opening
  • Deterministic — same file state always renders identically on all platforms
  • Self-contained — all decay state lives inside the file

File Format

.fmrl uses a PNG-style chunked binary format. Chunks are CRC-32 verified; unknown chunks are silently skipped for forward compatibility.

IHDRWidth, height, decay policy
DATA4-color palette + zlib-compressed tiles
AGE Per-tile: last_view, fade_level, noise_seed, edge_damage
METAOptional zlib-compressed JSON metadata
IENDTerminator

The AGE chunk is mutated in-place after every render: last_view and fade_level are updated and the chunk CRC is recomputed.
