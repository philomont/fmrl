FMRL

What is FMRL?

.fmrl (Fragile Manuscript Record Layer) is an image codec where visual degradation is a core design feature. Files age with every viewing — simulating the natural deterioration of physical media.

  • Temporal decay — pixels fade toward the paper background based on days since last view
  • Edge erosion — stroke edges wear away with each opening, reducing information rather than adding noise
  • Deterministic — same file state always renders identically across all platforms
  • Self-contained — all decay state lives inside the file's AGE chunk

How aging works

The Age button applies morphological erosion: any non-paper pixel with three or more paper 8-neighbors becomes paper, eroding one layer from every exposed surface per press. The eroded file compresses smaller because larger uniform regions are more compressible — information is genuinely lost, not obscured.

Auto aging uses a lighter erosion step (no short-run elimination) so high-rate aging feels fluid rather than abrupt. The .fmrl format stores images as a 4-color indexed palette (ink, paper, crimson, white) in 32×32 tiles, each zlib-compressed. The AGE chunk carries per-tile timestamps and erosion counters rewritten with every render.

Saving bakes in 10 erosion steps. Opening applies one more — images are noticeably older each time the file changes hands.
